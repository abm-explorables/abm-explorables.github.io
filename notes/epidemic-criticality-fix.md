# epidemic SIR の臨界バグ調査と修正 (2026-06-07)

「感染力の臨界点周りで全く創発がない」(Yuito) を実測調査して修正した記録。

## 症状
スライダーで感染力 β を上げても、相転移の劇的さ(sub で消える ↔ super で全域点火)が見えない。臨界マーカーの位置でも何も起きない。

## 調査(Node で sim を逐語再現してスイープ、各点 200〜400 試行)
2つの独立した病巣が判明:

1. **マーカー誤配置**: 元コードは閾値を平均場 `betaC = γ/meanDeg` で置いていた。だが空間ネットワーク(地理的近接グラフ)はクラスタリングで感染が無駄打ちされ、実際の流行閾値は平均場予測の約3倍高い。実測ではマーカー位置(スライダー19%)で outbreak 平均4%・takeoff率28% ── ほぼ何も起きない場所にマーカーがあった。

2. **転移の washed-out**: N=420 と小さく、γ=0.07 が高い。1辺あたり感染確率 T=β/(β+γ) が β を上げても頭打ち(slider max でも T=0.42)で全域に燃え広がらず、スライダー最大でも outbreak 56%・std 19%。相転移がボケて「全域一斉点火」のドラマが出ない。

## 修正(パラメータ + R0 定義)
- N: 420 → **700**(有限サイズ効果を抑え転移を鋭く)
- R(接続半径): 0.07 → **0.055**、γ: 0.07 → **0.035**(meanDeg≈8 を保ちつつ T のレンジを広げ、super で全域点火 ≈96% に)
- **R0 を平均場 → パーコレーション較正に**: `R0 = T / T_c × (1−v)`、T=β(1−d)/(β(1−d)+γ)、実測 **T_c = 0.241**。これで R0=1 が本当の転移点に一致する(平均場の β/γ×次数 は廃止)
- マーカー: 実測 β_c = T_c·γ/(1−T_c) ≈ **0.0111**(スライダー22%)に配置
- 既定 β: 0.022 → **0.010**(R0=0.92、sub 側から始めて上げて閾値を探す導線)
- foot の式記述も平均場 → パーコレーション T_c 越えの honest な記述に修正

## 修正後の挙動(実測)
takeoff 率が β=0.008→0.011→0.015→0.022 で 14%→49%→80%→93% と急上昇。sub で確実に消え、super で 90%+ が点火。臨界近傍で試行ごとに大きくぶれる(臨界のゆらぎ)。= 狙いどおりの可視的な相転移。

調査スクリプトは使い捨て(_sir_*.mjs、commit せず削除済み)。再調査が要れば epidemic.html / EpidemicSIR.astro の sim を Node に写して同様にスイープする。
